# 开发服务器 500 错误修复指南

## 问题描述
浏览器显示 500 Internal Server Error，无法加载样式表和静态资源。

## 原因分析
这不是数据库问题，而是 Next.js 开发服务器的问题：
1. 开发服务器崩溃或卡住
2. 多个 Node 进程冲突
3. .next 缓存损坏
4. 端口被占用

## 快速修复（3步）

### 步骤 1: 停止所有 Node 进程

**Windows (PowerShell):**
```powershell
Stop-Process -Name "node" -Force
```

**Windows (CMD):**
```cmd
taskkill /F /IM node.exe
```

**Mac/Linux:**
```bash
killall node
```

### 步骤 2: 清理缓存

```bash
# 删除 .next 文件夹
rm -rf .next

# 或者 Windows
rmdir /s /q .next
```

### 步骤 3: 重新启动开发服务器

```bash
npm run dev
```

等待看到：
```
✓ Ready in xxxx ms
- Local: http://localhost:3000
```

## 详细修复步骤

### 方法 1: 完全重启（推荐）

```bash
# 1. 停止所有 Node 进程
taskkill /F /IM node.exe

# 2. 清理缓存
rmdir /s /q .next

# 3. 重新安装依赖（可选）
npm install

# 4. 启动开发服务器
npm run dev
```

### 方法 2: 使用不同的端口

如果端口 3000 被占用：

```bash
# 使用端口 3001
PORT=3001 npm run dev

# Windows
set PORT=3001 && npm run dev
```

### 方法 3: 清理所有缓存

```bash
# 停止服务器
Ctrl + C

# 清理所有缓存
rm -rf .next
rm -rf node_modules/.cache

# 重新启动
npm run dev
```

## 常见问题

### 问题 1: 端口被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方案：**
```bash
# 找到占用端口的进程
netstat -ano | findstr :3000

# 杀死该进程（替换 PID）
taskkill /F /PID <PID>

# 或使用不同端口
set PORT=3001 && npm run dev
```

### 问题 2: 模块未找到

**错误信息：**
```
Module not found: Can't resolve 'xxx'
```

**解决方案：**
```bash
# 重新安装依赖
rm -rf node_modules
npm install
```

### 问题 3: 缓存问题

**错误信息：**
```
Error: ENOENT: no such file or directory
```

**解决方案：**
```bash
# 清理所有缓存
rm -rf .next
rm -rf node_modules/.cache
npm run dev
```

### 问题 4: 内存不足

**错误信息：**
```
JavaScript heap out of memory
```

**解决方案：**
```bash
# 增加 Node 内存限制
set NODE_OPTIONS=--max_old_space_size=4096
npm run dev
```

## 预防措施

### 1. 正确停止服务器

总是使用 `Ctrl + C` 停止服务器，不要直接关闭终端

### 2. 定期清理缓存

```bash
# 每周执行一次
rm -rf .next
npm run dev
```

### 3. 使用进程管理器

安装 PM2：
```bash
npm install -g pm2

# 启动
pm2 start npm --name "musicx" -- run dev

# 停止
pm2 stop musicx

# 重启
pm2 restart musicx
```

### 4. 监控内存使用

```bash
# 查看 Node 进程
tasklist | findstr node

# 查看内存使用
wmic process where name="node.exe" get ProcessId,WorkingSetSize
```

## 开发最佳实践

### 1. 使用 .env.local

创建 `.env.local` 文件：
```env
NEXT_PUBLIC_SUPABASE_URL=your_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_key
```

### 2. 启用快速刷新

在 `next.config.mjs` 中：
```javascript
const nextConfig = {
  reactStrictMode: true,
  swcMinify: true,
}
```

### 3. 使用开发工具

```bash
# 安装开发工具
npm install -D @next/bundle-analyzer

# 分析包大小
ANALYZE=true npm run build
```
