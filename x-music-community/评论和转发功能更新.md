# 评论和转发功能更新说明

## 修复的问题

### 1. 评论无法显示 ✅
**问题原因：**
- 查询使用了错误的字段名 `full_name`，但数据库中是 `nickname`
- 外键关联不明确

**解决方案：**
```typescript
// 修改前
.select(`
  *,
  user:profiles(username, full_name, avatar_url)
`)

// 修改后
.select(`
  *,
  user:profiles!comments_user_id_fkey(username, nickname, avatar_url)
`)
```

### 2. 转发功能重新设计 ✅
**原来的问题：**
- 转发只增加计数，不创建新帖子
- 用户看不到转发的内容

**新的设计：**
转发会创建一条新帖子，包含：
- 转发者的评论（可选）
- 原帖作者和内容的引用
- 原帖的媒体内容（图片/音频/视频）
- 同时记录转发关系到 retweets 表

**带评论的转发：**


**不带评论的转发：**
```
转发 @原作者: 原帖内容
[原帖的媒体内容]
```

#### 转发特点
- ✅ 创建新帖子，显示在首页

## 数据库说明

### comments 表
```sql
CREATE TABLE comments (
  id UUID PRIMARY KEY,
  post_id UUID REFERENCES posts(id),
  user_id UUID REFERENCES auth.users(id),
  content TEXT NOT NULL,
  created_at TIMESTAMP
);
```

### retweets 表
```sql
CREATE TABLE retweets (
  id UUID PRIMARY KEY,
  post_id UUID REFERENCES posts(id),  -- 原帖ID
  user_id UUID REFERENCES auth.users(id),  -- 转发者ID
  comment TEXT,  -- 转发评论（可选）
  created_at TIMESTAMP,
  UNIQUE(post_id, user_id)  -- 不能重复转发
);
```

### 外键关系
```
comments.user_id -> auth.users.id -> profiles.id
```

使用明确的外键名称：`comments_user_id_fkey`

## 常见问题

### Q: 评论不显示？
A: 
1. 检查浏览器控制台错误
2. 确认 profiles 表有 nickname 字段
3. 确认外键关系正确
4. 刷新页面重新加载

### Q: 转发后看不到新帖子？
A: 
1. 刷新首页
2. 检查是否有错误提示
3. 查看数据库 posts 表是否有新记录
4. 确认用户 profile 信息完整

### Q: 转发的媒体内容不显示？
A: 这是正常的，转发会复制原帖的媒体链接，确保原帖的媒体文件仍然可访问。

### Q: 可以取消转发吗？
A: 不能。转发会创建新帖子，如果要删除，需要删除转发的帖子（功能待实现）。

### Q: 可以转发自己的帖子吗？
A: 可以，但通常不建议这样做。

## 技术细节

### 为什么使用明确的外键名称？
```typescript
// 不明确（可能失败）
user:profiles(...)

// 明确（推荐）
user:profiles!comments_user_id_fkey(...)
```

Supabase 需要明确的外键名称来正确关联表。

### 转发为什么创建新帖子？
这样设计的好处：
1. 转发内容出现在首页时间线
2. 可以对转发帖子进行点赞、评论
3. 增加内容的传播性
4. 符合社交媒体的使用习惯
