### 2. 类型错误 ✅
**问题：** 
- `comment.user?.avatar_url` 可能为 undefined
- `profile.avatar_url` 可能为 undefined
- Profile 类型字段不是可选的

**解决方案：**
```typescript
// 修改前
interface Profile {
  avatar_url: string
  posts_count: number
  following_count: number
  followers_count: number
}

// 修改后
interface Profile {
  avatar_url?: string
  posts_count?: number
  following_count?: number
  followers_count?: number
}
```

## 技术细节

### 数据库触发器
自动更新统计数据，无需手动维护：

```sql
-- 关注时自动更新
CREATE TRIGGER on_follow_created
  AFTER INSERT ON follows
  FOR EACH ROW EXECUTE FUNCTION update_user_stats();

-- 取消关注时自动更新
CREATE TRIGGER on_follow_deleted
  AFTER DELETE ON follows
  FOR EACH ROW EXECUTE FUNCTION update_user_stats_on_delete();
```

### RLS 策略
确保数据安全：

```sql
-- 用户只能关注/取消关注自己的操作
CREATE POLICY "Users can follow others" ON follows
FOR INSERT WITH CHECK (auth.uid() = follower_id);

CREATE POLICY "Users can unfollow others" ON follows
FOR DELETE USING (auth.uid() = follower_id);
```

### TypeScript 类型安全
使用可选字段和默认值：

```typescript
// 类型定义
interface Profile {
  posts_count?: number
}

// 使用
{profile.posts_count || 0}
```
