# 功能测试指南

## 评论功能测试

### 测试步骤

1. **查看评论**
   - 找到任意帖子
   - 点击💬图标
   - 评论区应该展开
   - 如果有评论，应该显示列表

2. **发表评论**
   - 在评论输入框输入："这个很棒！"
   - 点击发送按钮（✈️图标）
   - 评论应该立即出现在列表顶部
   - 评论数应该 +1

3. **验证评论显示**
   - 评论应该显示：
     - 评论者头像
     - 评论者昵称
     - 发表时间（如"刚刚"、"2小时前"）
     - 评论内容

### 预期结果
```
[头像] 你的昵称 · 刚刚
       这个很棒！
```

### 如果评论不显示

**检查浏览器控制台：**
```javascript
// 应该看到类似的请求
POST /rest/v1/comments
{
  "post_id": "...",
  "user_id": "...",
  "content": "这个很棒！"
}
```

**检查数据库：**
```sql
-- 查看评论是否保存
SELECT * FROM comments 
ORDER BY created_at DESC 
LIMIT 5;

-- 检查 profiles 表
SELECT id, username, nickname, avatar_url 
FROM profiles 
LIMIT 5;
```

**常见问题：**
1. profiles 表缺少 nickname 字段 → 执行 `fix-database.sql`
2. 外键关系错误 → 检查 comments 表的 user_id 外键
3. 权限问题 → 检查 RLS 策略

## 转发功能测试

### 测试步骤

1. **转发帖子（不带评论）**
   - 找到想转发的帖子
   - 点击🔄图标
   - 弹出转发对话框
   - 直接点击"转发"按钮（不输入评论）
   - 应该显示"转发成功！"提示

2. **查看转发结果**
   - 返回首页（刷新页面）
   - 应该看到新的转发帖子
   - 格式：`转发 @原作者: 原帖内容`
   - 如果原帖有媒体，转发帖子也应该显示

3. **转发帖子（带评论）**
   - 找到另一个帖子
   - 点击🔄图标
   - 在对话框输入："强烈推荐！"
   - 点击"转发"按钮
   - 返回首页查看

4. **验证转发显示**
   - 转发帖子应该显示：
     ```
     强烈推荐！
     
     转发 @原作者: 原帖内容
     [原帖的图片/音频/视频]
     ```

5. **检查统计数据**
   - 原帖的转发数应该 +1
   - 你的发帖数应该 +1
   - 转发按钮应该变成绿色

### 预期结果

**不带评论的转发：**
```
转发 @电音制作人Alex: 刚刚完成了一首新的电子音乐作品...
[音频播放器]
```

**带评论的转发：**
```
强烈推荐！

转发 @电音制作人Alex: 刚刚完成了一首新的电子音乐作品...
[音频播放器]
```

### 如果转发失败

**检查浏览器控制台：**
```javascript
// 应该看到两个请求

// 1. 创建新帖子
POST /rest/v1/posts
{
  "content": "强烈推荐！\n\n转发 @...",
  "user_id": "...",
  ...
}

// 2. 记录转发关系
POST /rest/v1/retweets
{
  "post_id": "...",
  "user_id": "...",
  "comment": "强烈推荐！"
}
```

**检查数据库：**
```sql
-- 查看转发记录
SELECT * FROM retweets 
ORDER BY created_at DESC 
LIMIT 5;

-- 查看转发创建的帖子
SELECT * FROM posts 
WHERE content LIKE '转发 @%'
ORDER BY created_at DESC 
LIMIT 5;

-- 检查用户 profile
SELECT id, username, nickname, avatar_url 
FROM profiles 
WHERE id = 'YOUR_USER_ID';
```

**常见问题：**
1. 无法获取用户信息 → 检查 profiles 表是否有你的记录
2. 转发数不更新 → 检查触发器是否创建
3. 重复转发 → retweets 表有 UNIQUE 约束，不能重复转发

## 数据库验证

### 检查所有表
```sql
-- 检查表是否存在
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('posts', 'comments', 'retweets', 'post_likes', 'follows', 'profiles');
```

### 检查字段
```sql
-- 检查 profiles 表字段
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'profiles';

-- 检查 posts 表字段
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'posts';
```

### 检查触发器
```sql
-- 查看所有触发器
SELECT trigger_name, event_object_table, action_statement
FROM information_schema.triggers
WHERE trigger_schema = 'public';
```

### 检查数据
```sql
-- 查看最新的帖子
SELECT id, content, author_name, likes_count, comments_count, retweets_count
FROM posts
ORDER BY created_at DESC
LIMIT 10;

-- 查看最新的评论
SELECT c.content, p.nickname as commenter
FROM comments c
LEFT JOIN profiles p ON c.user_id = p.id
ORDER BY c.created_at DESC
LIMIT 10;

-- 查看最新的转发
SELECT r.comment, p.nickname as retweeter
FROM retweets r
LEFT JOIN profiles p ON r.user_id = p.id
ORDER BY r.created_at DESC
LIMIT 10;
```
