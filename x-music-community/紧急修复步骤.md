# 紧急修复步骤

## 问题：登录后页面崩溃，500 错误

### 立即执行（5分钟解决）

#### 步骤 1: 执行诊断和修复脚本

1. 打开 Supabase Dashboard
2. 进入 SQL Editor
3. 复制 `诊断和修复脚本.sql` 的全部内容
4. 粘贴并点击 "Run"
5. 等待执行完成

**这个脚本会：**
- ✅ 检查所有用户
- ✅ 找出缺少 profile 的用户
- ✅ 自动创建缺失的 profile
- ✅ 修复所有数据
- ✅ 验证修复结果

#### 步骤 2: 清除浏览器缓存

```
Windows: Ctrl + Shift + Delete
Mac: Cmd + Shift + Delete
```

选择：
- ✅ Cookie 和其他网站数据
- ✅ 缓存的图片和文件

点击"清除数据"

#### 步骤 3: 硬刷新页面

```
Windows: Ctrl + F5
Mac: Cmd + Shift + R
```

#### 步骤 4: 重新登录

1. 访问登录页面
2. 输入账号密码
3. 点击登录
4. 应该正常显示

```sql

```

## 预防措施

### 1. 确保触发器正常工作

```sql
-- 检查触发器
SELECT trigger_name, event_object_table
FROM information_schema.triggers
WHERE trigger_schema = 'public'
AND event_object_table IN ('users', 'profiles');

-- 应该看到 on_auth_user_created 触发器
```

### 2. 测试新用户注册

1. 注册一个新账号
2. 立即检查是否有 profile：
   ```sql
   SELECT * FROM profiles 
   WHERE username = 'NEW_USERNAME';
   ```
3. 应该自动创建

### 3. 定期检查数据完整性

```sql
-- 每周执行一次
SELECT 
  COUNT(*) as total_users,
  COUNT(p.id) as users_with_profile
FROM auth.users u
LEFT JOIN profiles p ON u.id = p.id;

-- total_users 应该等于 users_with_profile
```
